# Backup Environment — Guidelines for Codex / Copilot

## 1. Общая задумка проекта

Мы делаем **локальное версионное хранилище бэкапов**, которое:

- хранит файлы в собственной структуре (`objects/` + метаданные),
- работает **стримингово**, без создания временных полных копий больших файлов,
- позже умеет **прозрачное шифрование** объектов,
- защищается через **локальную системную аутентификацию** (macOS Touch ID / пароль учётки),
- поверх этого появится **удобный UI**, но всё ядро должно быть доступно через консоль.

Сейчас проект нацелен **на macOS**, но в архитектуре надо избегать жёстких платформенных завязок, чтобы в будущем можно было портировать.

---

## 2. Структура хранилища (storage layout)

По умолчанию корень хранилища:

`~/Library/Application Support/gs-backup-storage/`

Может быть переопределён:

`GSBACKUP_STORAGE=/кастомный/путь`

Структура:

```
<storage_root>/
    config.json
    meta/
        index.json
    objects/
        <sha256>
```

Объекты пока хранятся незашифрованными.

## 3. Основные модули проекта

### 3.1 storage_paths
Определяет путь к хранилищу.

### 3.2 storage_init
Создаёт структуру каталогов и базовые файлы.

### 3.3 index
Чтение/запись index.json, управление версиями.

### 3.4 objects
Стриминговое чтение/запись объектов, подсчёт SHA-256.

### 3.5 crypto (в будущем)
Шифрование / дешифрование, два хэша, KDF, recovery key.

### 3.6 auth_mac (в будущем)
Touch ID / пароль учётки macOS.

### 3.7 CLI
Команды init, add, list, verify, restore, remove, stats.

---

## 4. Вехи проекта

### Веха 1 — CLI + локальное хранилище
- init, add, list, verify, restore.
- Стриминговая работа с файлами.
- Атомарная запись index.json.

### Веха 2 — UI
TUI или GUI поверх ядра.

### Веха 3 — Шифрование
AES-GCM, два хэша, KDF, совместимость со старыми объектами.

### Веха 4 — Touch ID
Защита доступа к хранилищу локальной аутентификацией.

---

## 5. Общие правила разработки

- Python 3.x + type hints.
- Только стриминговая обработка больших файлов.
- Время — UTC, timezone-aware.
- Атомарные обновления индекса.
- Модульность: CLI → вызывает core, core не знает о CLI.
- Минимальные зависимости.
